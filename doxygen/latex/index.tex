 \mbox{\hyperlink{namespaceEBGeometry}{EBGeometry}} is a code for


\begin{DoxyEnumerate}
\item Turning watertight and orientable surface grids into signed distance functions (SDFs).
\item Fast evaluation of such grids using bounding volume hierarchies (BVHs).
\item Providing fast constructive solid geometry (CSG) unions using BVHs.
\end{DoxyEnumerate}

This code is header-\/only and can be dropped into any C++ project that supports C++14. It was originally written to be used with embedded-\/boundary (EB) codes like Chombo or AMReX. However, \mbox{\hyperlink{namespaceEBGeometry}{EBGeometry}} provides quite general SDFs, implicit functions, and CSG unions, and is useful beyond those codes.

To clone \mbox{\hyperlink{namespaceEBGeometry}{EBGeometry}}\+: \begin{DoxyVerb}git clone git@github.com:rmrsk/EBGeometry.git
\end{DoxyVerb}
 \doxysection*{Requirements}


\begin{DoxyItemize}
\item A C++ compiler which supports C++14.
\end{DoxyItemize}

\mbox{\hyperlink{namespaceEBGeometry}{EBGeometry}} is a header-\/only library in C++ and has no external dependencies. To use it, simply make \mbox{\hyperlink{EBGeometry_8hpp_source}{EBGeometry.\+hpp}} visible to your code and include it.

\doxysection*{Example quickstart}

Several examples are given in the Examples folder. To run one of the examples, navigate to the example and compile and run it.

\doxysubsection*{1. Analytic distance fields}

Use one of the analytic signed distance fields in \mbox{\hyperlink{namespaceEBGeometry}{EBGeometry}}, and rotate and transform it.


\begin{DoxyCode}{0}
\DoxyCodeLine{cd Examples/EBGeometry\_Shapes}
\DoxyCodeLine{g++ -\/O3 -\/std=c++14 main.cpp}
\DoxyCodeLine{./a.out}

\end{DoxyCode}


\doxysubsection*{2. Creating an SDF from a surface grid}

Read in an STL file and turn it into a signed distance field.


\begin{DoxyCode}{0}
\DoxyCodeLine{cd Examples/EBGeometry\_DCEL}
\DoxyCodeLine{g++ -\/O3 -\/std=c++14 main.cpp}
\DoxyCodeLine{./a.out armadillo.stl}

\end{DoxyCode}




 

~\newline


\doxysubsection*{3. Fast CSG operations for analytic functions}

Generate a spherical packed bed geometry using analytic representations of spheres. BVH-\/accelerated CSG unions lead to order-\/of-\/magnitude faster implicit function evaluation. 
\begin{DoxyCode}{0}
\DoxyCodeLine{cd Examples/EBGeometry\_Union}
\DoxyCodeLine{g++ -\/O3 -\/std=c++14 main.cpp}
\DoxyCodeLine{./a.out}

\end{DoxyCode}




 

~\newline


\doxysubsection*{4. Fast CSG on composite geometries}

Create an F18 implicit function using CSG on each part (65 parts in total).


\begin{DoxyCode}{0}
\DoxyCodeLine{cd Examples/EBGeometry\_F18}
\DoxyCodeLine{g++ -\/O3 -\/std=c++17 main.cpp -\/lstdc++fs}
\DoxyCodeLine{./a.out}

\end{DoxyCode}




 

~\newline


\doxysubsection*{5. Advanced examples}

Some complex examples that use Chombo3 or AMReX are also provided in the Examples folder. These include application-\/specific code and require installation of Chombo3 or AMReX.

For more advanced usage, users can supply their own file parsers, provide their own bounding volumes, or their own \mbox{\hyperlink{namespaceBVH}{BVH}} partitioners. \mbox{\hyperlink{namespaceEBGeometry}{EBGeometry}} is not too strict about these things, and uses rigorous templating for ensuring that the \mbox{\hyperlink{namespaceEBGeometry}{EBGeometry}} functionality can be extended.
